TODO: pipeline architecture summary and next work

Current capabilities:
- Stage-based CLI with `download`, `bidsify`, `validate`, `qc`, `exclude`, `preprocess`, `run`, and `status`.
- Per-subject JSON state file in `sub-XXX/.pipeline_state.json`.
- `PipelineRunner` determines next stage and skips completed stages.
- `hbicproc run sub-XXX` resumes a single subject from its next incomplete stage.
- `hbicproc run --all` resumes all subjects found in the BIDS output directory.
- `hbicproc status` generates an SVG completion grid for subjects/sessions vs stages.
- `--rerun` support for stage commands to override completion state.
